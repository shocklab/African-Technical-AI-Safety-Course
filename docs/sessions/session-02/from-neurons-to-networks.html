🧮 From Neurons to Networks

A fast, maths-honest recap: what a network computes, and what training actually does

🧮 Week 2 — deep learning & scaling laws

To reason about how these systems fail, you need a working picture of how they're built. This session rebuilds that picture at the level a mathematician wants — functions, parameters, gradients — then shows why capability scales so predictably with size.

This session's sub-sessions: 2.1 from neurons to networks · 2.2 inside the transformer · 2.3 neural scaling laws · 2.4 pace of progress & the emergence debate · 2.5 lab (Colab).

🎯 What we'll cover

You have met neural networks before, but cohorts vary, so we set a common, precise baseline. This sub-session frames a network as a parametrised function fit by gradient descent — nothing mystical — and pins down the few ideas (the loss, the gradient, the objective) that the safety arguments later hang on.

📐 A network is a parametrised function

Strip away the biology metaphor and a neural network is just a function fθ : ℝn → ℝm with a great many parameters θ.

From a neuron to a layer

Start with a single neuron: it takes an input vector x ∈ ℝdᵢₙ, forms a weighted sum with its own weight vector w and bias b, and passes the result through a nonlinearity — producing one scalar, a = σ(w·x + b). Now stack dout neurons side by side and collect their weight vectors as the rows of a matrix W. The whole layer then computes a vector in one shot:

h = σ(Wx + b),   W ∈ ℝdout×dᵢₙ,  b, h ∈ ℝdout

Here σ is applied elementwise — the same scalar function to each component — and the standard choice is ReLU, σ(z) = max(0, z). A deep network chains L such layers, each feeding the next:

h(ℓ) = σ(W(ℓ) h(ℓ−1) + b(ℓ)),   h(0) = x

and the full collection of weights and biases {W(ℓ), b(ℓ)} is exactly what we bundled into the parameters θ. Without σ the composition of linear maps would collapse to a single linear map — the nonlinearity is what lets depth buy expressive power. By the universal-approximation results such a network can approximate essentially any function; the real engineering question is whether gradient descent can find parameters that do so.

🎯 Training = minimising a loss

"Learning" is optimisation. We choose a loss that scores predictions against targets and descend it.

The loop, precisely

  • Loss. A scalar L(θ) measuring how wrong the model is on data — for language models, the cross-entropy of the predicted next-token distribution against the true token.
  • Gradient. Compute ∇θL by back-propagation (the chain rule, applied layer by layer).
  • Step. Update θ ← θ − η ∇θL for a small learning rate η (in practice, a variant like Adam). Each step estimates the gradient on a mini-batch — a small random subset of the data, far cheaper than using all of it — and we repeat for a very long time.

That is the whole of it. Everything a frontier model "knows" is the low-loss configuration of θ this process settles into on its training data.

💡 What the language-model loss actually is

A language model is trained to predict the next token — a chunk of text, defined precisely in Session 2.2 — given the tokens before it. At each position it outputs a probability distribution pθ over the whole vocabulary (the softmax of its output scores; softmax is defined in 2.2), and the loss is the cross-entropy:

L = −(1/T) Σi log pθ(ti | t<i)

where ti is the true token at position i and t<i all tokens before it. (General cross-entropy is −Σ ptrue log pθ; because the true "distribution" puts all its mass on the one token that actually occurred, the sum collapses to the negative log-probability of that token.) So minimising L just means making the correct token as probable as possible. Two things to carry forward: this is exactly the "loss" that falls as a power law in Session 2.3; and the model is never told how to be right, only scored on the probability it placed on what actually came next.

⚠️ Why this framing matters for safety

🎯 We get to choose the loss — not the behaviour

This is the hinge the rest of the course turns on. We do not program behaviour; we specify a loss and an optimiser, and behaviour is whatever minimises that loss on that data. The model optimises the objective we wrote down, which is only ever a proxy for what we meant. Every alignment failure you'll study is, at root, a gap between those two — surfaced by an optimiser doing exactly its job.

💡 Two consequences to keep

First, the objective is a specification problem: get the loss subtly wrong and you get subtly wrong behaviour, at scale (Session 3, Goodhart). Second, the parameters are opaque: gradient descent finds a solution, not an interpretable one — which is exactly why we need mechanistic interpretability (Weeks 7–8) to read back out what the network actually learned.
