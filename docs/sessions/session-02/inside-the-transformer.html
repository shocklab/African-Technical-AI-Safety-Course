Week 2 • Session 2.2

🔬 Inside the Transformer

Tokens, attention, and the residual stream — the architecture, at the level interpretability needs

🎯 What we'll cover

Every model in this course is a transformer. This sub-session builds it properly — not as a black box but as a sequence of linear-algebraic operations you could, in principle, trace by hand. We pay special attention to attention and to the residual stream, because that picture is the one mechanistic interpretability (Weeks 7–8) literally reads off.

🔤 From text to vectors

Tokens and embeddings

  • Tokenisation. Text is split into tokens (sub-word pieces) from a fixed vocabulary of size V (tens of thousands). "tokenisation" might become token | isation.
  • Embedding. Each token id is mapped to a vector in ℝd via a learned matrix WE ∈ ℝV×d. A length-T sequence becomes a T×d matrix.
  • Position. Because attention is permutation-equivariant, positional information is added so the model knows token order.

👁️ Attention: routing information between positions

Attention is the one genuinely new idea. It lets each position pull in information from other positions, weighted by relevance.

Scaled dot-product attention

From the residual vectors, three learned projections produce queries, keys and values: Q = XWQ, K = XWK, V = XWV. The output is

Attention(Q,K,V) = softmax( QKᵀ / √dk ) V

Read it as: each position forms a query, compares it (dot product) against every position's key to get a distribution of attention weights, and reads out a weighted sum of values. In a language model the softmax is causally masked so a position can attend only to earlier ones — that's what makes next-token prediction well-posed. Several such heads run in parallel, each free to learn a different routing pattern (one of which, the induction head, you'll reverse-engineer in Week 7).

🌊 The residual stream

🌊 The picture to remember

A transformer block doesn't replace its input — it adds to it: x ← x + Attention(x), then x ← x + MLP(x). So every position carries a running vector — the residual stream — and each attention head and MLP reads from it and writes a contribution back. The stream is a shared communication channel that information is passed along and accumulated in, layer by layer, until the final vector is unembedded into a distribution over the next token.

This additive, read/write view is not a metaphor — it's the literal computation, and it is the foundation of mechanistic interpretability: "finding a circuit" means identifying which heads write what into the stream, and which later components read it.

💡 The MLP and the unembedding

Between attention sub-layers, a position-wise MLP (two linear layers with a nonlinearity) does most of the network's "computation per token" and holds much of its stored knowledge. At the end, a learned unembedding WU maps the final residual vector to V logits; a softmax turns those into the predicted next-token distribution. Training minimises the cross-entropy of that distribution against the true next token — the loss from Session 2.1.

🧷 Why the details earn their keep

Tokenisation has consequences

African and other low-resource languages are often tokenised into far more pieces than English, raising cost and degrading quality — one concrete root of the safety and capability gaps this course returns to.

Attention is inspectable

Because attention weights and head outputs are explicit, we can watch the model route information — the entry point for interpretability and for catching deceptive or unexpected behaviour.

Next-token prediction is the whole objective

Everything — reasoning, code, refusal — is learned in service of predicting the next token on the training data. Keep asking what behaviour that objective does and doesn't incentivise.
