Activation patching & causal interventions

The logic of "patch a clean activation into a corrupted run and see what recovers" — causal mediation made operational, and its cheap linear approximation

What we'll cover

Session 13 found components by reading weights. This sub-session adds the technique that turns "this component correlates with the behaviour" into "this component causes it": activation patching. The idea is causal-mediation analysis on the network's own computation graph — run the model clean, run it corrupted, then restore one internal activation and measure how much of the clean answer comes back. We write the metric down, distinguish the two directions (noising and denoising), and derive attribution patching, the first-order approximation that estimates every patch at once from a single backward pass. We also stay honest about what a successful patch licenses you to claim, which is less than it first appears.

Patching is causal mediation on the computation graph

A trained network is a fixed causal graph you can intervene on at will — that is a luxury no field studying brains has.

The setup needs a pair of inputs that differ in the thing you care about. Take ROME's example: a clean prompt ("The Space Needle is in downtown ___" → "Seattle") and a corrupted version where the subject tokens are damaged (Gaussian noise added to the "Space Needle" embeddings) so the model can no longer answer. Three runs define the experiment:

The three runs and the indirect effect

  • Clean: run the clean prompt, record the probability of the correct token, \(P_*[o]\) high.
  • Corrupted: run the corrupted prompt; the correct-token probability collapses, \(P_*[o]\) low.
  • Corrupted-with-restoration: run the corrupted prompt, but force one internal activation \(h_i^{(\ell)}\) (the hidden state at layer \(\ell\), token \(i\)) back to the value it had in the clean run, and let everything else stay corrupted.

The indirect effect of that activation is how much restoring it alone recovers the answer:

\[ \mathrm{IE}(h_i^{(\ell)}) \;=\; P_{*,\,\text{clean }h_i^{(\ell)}}[o] \;-\; P_*[o], \]

where \(P_*\) is the fully-corrupted probability. Average over many prompts to get the average indirect effect (AIE). Sweep \((\ell, i)\) over the whole grid and you get a heat-map of which activations carry the information. ROME's central finding came straight from this: two causal sites in GPT-2-XL, an expected "late site" at the last token and a newly-found "early site" at the last subject token in middle-layer MLPs, with the MLP contribution there far exceeding attention's. They then showed the located fact could be edited with a closed-form rank-one update to that MLP — an instance of "interpretability claim → causal test → intervention."

Noising, denoising, and what you can patch

The direction of the patch changes the question you are asking.

There are two ways to run the intervention, and they answer subtly different questions. Denoising (ROME's version) restores a clean activation into a corrupted run and asks "is this activation sufficient to recover the behaviour?" Noising patches a corrupted activation into a clean run and asks "is this activation necessary: does damaging it break the behaviour?" You can patch at different granularities — a whole residual-stream vector, a single MLP output, one attention head's contribution, even one head at one position — and the granularity decides how sharply you localise. A common refinement uses a symmetric counterfactual pair (two real prompts that differ minimally, rather than Gaussian noise) so the corruption is in-distribution; this matters because off-distribution corruption can produce effects that say more about the corruption than the circuit.

What a successful patch does and does not license

A patch that recovers the behaviour shows the activation is sufficient to restore it, on this distribution, under this corruption. It does not show the component's "purpose," that it is the only route, or that the story transfers to other inputs. Backup behaviour (Session 15.2) is the standing warning: ablate the component that patching fingered and a different one can silently take over, so "necessary" and "sufficient" both need care. Treat a patching heat-map as a strong localisation hypothesis to be confirmed by ablation and multiple distributions, not as a proof of mechanism. This is the same evidence-grading discipline from Session 13.1, now applied to a specific tool.

Attribution patching: every patch at once, for the price of one

Exact patching costs one forward pass per component. A first-order Taylor expansion costs one backward pass for all of them.

Exact activation patching is expensive: to test \(N\) components you run \(N\) separate forward passes. For a frontier model with thousands of components per layer that is prohibitive. Attribution patching (Nanda) is the cheap approximation. Write the metric \(L\) as a function of an activation \(a\); the effect of changing \(a\) from its corrupted value \(a_\text{corrupt}\) to its clean value \(a_\text{clean}\) is, to first order,

\[ \Delta L \;\approx\; \big(a_\text{clean} - a_\text{corrupt}\big)^\top \left.\frac{\partial L}{\partial a}\right|_{a=a_\text{corrupt}}. \]

This is a Taylor expansion of the true patch effect around the corrupted run. The gradient \(\partial L/\partial a\) for every activation in the network is obtained from a single backward pass (plus the two forward passes for the clean and corrupted activations), so you estimate all \(N\) patch effects at once instead of running \(N\) forwards. It is an approximation — it assumes the effect is roughly linear in the activation over the clean-to-corrupted gap, which fails for large or saturating changes — so the workflow is: attribution-patch to cheaply rank the whole model, then exact-patch the top candidates to confirm. One line on the rigorous end of the spectrum: causal scrubbing (Redwood) formalises a circuit hypothesis as a claim about which activations are irrelevant, and tests it by resampling exactly those; if the behaviour survives the resampling, the hypothesis holds. Attribution patching is the cheap scout; causal scrubbing is the strict auditor.

Worked: trace one patch by hand

Take a two-layer toy model and the IOI-style metric (15.2) of logit difference between two candidate answers. Corrupt the input by swapping the two names. (i) Compute the clean and corrupted logit differences. (ii) Restore the layer-1 attention output at the final position to its clean value; recompute the logit difference; the recovery is that component's indirect effect. (iii) Now attribution-patch: with the corrupted run's cached activations and one backward pass of the logit-difference metric, estimate the same component's effect as \((a_\text{clean}-a_\text{corrupt})^\top \partial L/\partial a\), and compare to the exact value from (ii). Students see both the causal logic and where the linear approximation is tight versus loose, the judgement the technique demands in practice.

Questions to bring to class

  • Write the indirect-effect metric and explain why you need a pair of inputs (clean and corrupted) rather than one. What does a symmetric counterfactual buy over Gaussian-noise corruption?
  • Distinguish denoising ("sufficient") from noising ("necessary"). Give a circuit where a component is sufficient but not necessary, and say which run would reveal that.
  • Derive the attribution-patching estimate as a first-order Taylor expansion. Why does one backward pass give the effect for every component at once, and when is the approximation untrustworthy?
  • A patch recovers the behaviour. List three things this does not establish about the component, and say how you would check each.
